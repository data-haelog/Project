### TensorFlow vs PyTorch Performance Comparison Results

#### 실험 개요
- **데이터셋**: Fashion 이미지 861개
- **모델**: ResNet50 (ImageNet 사전학습)
- **임베딩 차원**: 2048

### 성능 비교 결과

#### 1. 처리 속도
| 프레임워크 | 처리 시간 | 이미지당 평균 시간 |
|-----------|----------|-----------------|
| TensorFlow | 88.35초 | 0.103초/이미지 |
| PyTorch | 68.44초 | 0.079초/이미지 |
| **속도 향상** | **-22.5%** | **PyTorch가 약 1.29배 빠름** |

#### 2. 임베딩 특성
| 프레임워크 | 평균값 | 특징 |
|-----------|--------|------|
| TensorFlow | 0.412421 | 상대적으로 높은 활성화 |
| PyTorch | 0.375098 | 상대적으로 낮은 활성화 |
| **차이** | **0.037323** | **약 9% 차이** |

### 주요 인사이트

####  PyTorch의 우수성
1. **처리 속도**: PyTorch가 약 20초(22.5%) 더 빠름
   - 배치 처리 최적화가 더 효율적
   - CUDA 연산 효율성이 높음

2. **GPU 활용**: PyTorch의 동적 그래프가 GPU 메모리 관리에 유리
   - 메모리 정리(gc.collect)와 캐시 정리가 효과적

####  임베딩 값 차이
1. **평균값 차이 (9%)**:
   - 전처리 방식의 미세한 차이
   - 정규화 파라미터 적용 순서 차이
   - 두 프레임워크 모두 ImageNet 표준 사용하지만 구현 방식이 다름

2. **상관관계 확인 필요**:
   - 평균값 차이가 있지만, 벡터 간 상관관계가 높다면 문제없음
   - Pearson 상관계수 0.9 이상이면 실질적으로 동일한 특성 추출

### 결론 및 권장사항

####  권장 프레임워크: **PyTorch**
**이유:**
1. **속도**: 22.5% 더 빠른 처리 속도
2. **확장성**: 대규모 데이터셋 처리 시 시간 절약 효과 증가
3. **유연성**: 모델 커스터마이징이 더 용이

####  추가 검증 필요 사항
1. **상관관계 분석**: 샘플별 Pearson 상관계수 확인
2. **유사도 테스트**: 실제 추천 결과 비교
3. **대용량 테스트**: 전체 데이터셋(44,000+)으로 성능 재검증

####  최종 선택 기준
- **속도 우선**: PyTorch 선택
- **안정성 우선**: 두 프레임워크 상관관계 확인 후 결정
- **실무 적용**: 추천 시스템 정확도로 최종 판단
